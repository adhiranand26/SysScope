A — Retro Terminal / CRT

Phosphor green on black. Scanlines. Monospace everything. Feels like SSH'ing into a mainframe from the 80s. Bold, nerdy, unforgettable.

Pros

  • Instantly distinctive — no one else will have this
  • Perfect thematic fit (Linux, sysadmin, OS internals)
  • CRT effects + scanlines = memorable presentation

Cons

  • Green-on-black can strain eyes over time
  • Harder to show complex charts (limited color palette)

B — Dark Cyberpunk / Neon

Deep indigo/violet background with electric cyan + purple accents. Glowing edges, neon data. Feels like a sci-fi control room. Modern, striking, premium.

Pros

  • Supports rich color coding for different data types
  • Modern + visually impressive for presentations
  • Easy to differentiate CPU/MEM/Disk with distinct neon hues

Cons

  • More common "dark dashboard" aesthetic (needs execution to stand out)
  • Can feel overdesigned if not restrained

C — Blueprint / Engineering Schematic

Deep navy with blueprint grid. Dashed borders, monospace labels, schematic annotation style. Feels like reading engineering drawings of your OS. Academic, technical, deeply thematic.

Pros

  • Perfectly fits "OS internals" educational context
  • Blueprint grid + dashed borders = unique, never seen in dashboards
  • Professor will see engineering intentionality, not just a "pretty UI"

Cons

  • Monospace-heavy can feel dense
  • Needs careful spacing to stay readable
